🐳 Docker로 NiFi 실습환경 구축

Docker를 활용한 빠르고 효율적인 Apache NiFi 학습 환경 구성 가이드

📋 개요

Docker를 사용하면 복잡한 설치 과정 없이 즉시 NiFi 실습 환경을 구축할 수 있습니다. 이 문서에서는 단일 노드부터 클러스터 환경까지 다양한 Docker 기반 NiFi 설정 방법을 다룹니다.

🔧 사전 준비사항

Docker 설치

Docker Desktop 또는 Docker Engine이 설치되어 있어야 합니다.

최소 버전: Docker 20.10+

시스템 요구사항

RAM: 최소 4GB (권장 8GB)

디스크: 최소 10GB 여유 공간

네트워크 포트

8443: NiFi Secure Web UI (HTTPS)

💡 팁: Docker Desktop을 사용하는 경우, Docker 설정에서 리소스 할당을 충분히 늘려주세요.

🚀 단일 노드 NiFi 실행

NiFi 이미지 Pull

# 최신 NiFi 이미지 다운로드
docker pull apache/nifi:latest

# 특정 버전 다운로드 (권장)
docker pull apache/nifi:1.24.0

기본 NiFi 컨테이너 실행

# 기본 실행 (HTTPS - NiFi 2.x부터 기본)
docker run --name nifi \
  -p 8443:8443 \
  -e SINGLE_USER_CREDENTIALS_USERNAME=admin \
  -e SINGLE_USER_CREDENTIALS_PASSWORD=adminPassword123! \
  -d \
  apache/nifi:latest
✅ 접속 URL: https://localhost:8443/nifi
약 30초 ~ 1분 후 접속 가능합니다.
로그인 정보: admin / adminPassword123!

고급 보안 설정

# 인증서와 함께 HTTPS 실행
docker run --name secure-nifi \
  -p 8443:8443 \
  -e SINGLE_USER_CREDENTIALS_USERNAME=admin \
  -e SINGLE_USER_CREDENTIALS_PASSWORD=StrongPassword123! \
  -e NIFI_WEB_HTTPS_PORT=8443 \
  -e NIFI_SENSITIVE_PROPS_KEY=my-random-string-at-least-12-chars \
  -d \
  apache/nifi:latest
⚠️ 주의: 프로덕션 환경에서는 더 강력한 보안 설정과 인증서를 사용하세요.

볼륨 마운트를 통한 데이터 영속성

# 호스트 디렉토리 생성
mkdir -p ~/nifi/database_repository
mkdir -p ~/nifi/flowfile_repository
mkdir -p ~/nifi/content_repository
mkdir -p ~/nifi/provenance_repository
mkdir -p ~/nifi/state
mkdir -p ~/nifi/logs
mkdir -p ~/nifi/conf

# 볼륨 마운트와 함께 실행
docker run --name nifi-persistent \
  -p 8443:8443 \
  -e SINGLE_USER_CREDENTIALS_USERNAME=admin \
  -e SINGLE_USER_CREDENTIALS_PASSWORD=adminPassword123! \
  -v ~/nifi/database_repository:/opt/nifi/nifi-current/database_repository \
  -v ~/nifi/flowfile_repository:/opt/nifi/nifi-current/flowfile_repository \
  -v ~/nifi/content_repository:/opt/nifi/nifi-current/content_repository \
  -v ~/nifi/provenance_repository:/opt/nifi/nifi-current/provenance_repository \
  -v ~/nifi/state:/opt/nifi/nifi-current/state \
  -v ~/nifi/logs:/opt/nifi/nifi-current/logs \
  -v ~/nifi/conf:/opt/nifi/nifi-current/conf \
  -d \
  apache/nifi:latest

🌐 Docker Compose를 활용한 환경 구성

단일 노드 Docker Compose 설정

# docker-compose.yml
version: '3.8'

services:
  nifi:
    image: apache/nifi:1.24.0
    container_name: nifi
    ports:
      - "8443:8443"
    environment:
      - NIFI_WEB_HTTPS_PORT=8443
      - SINGLE_USER_CREDENTIALS_USERNAME=admin
      - SINGLE_USER_CREDENTIALS_PASSWORD=adminPassword123!
      - NIFI_SENSITIVE_PROPS_KEY=my-sensitive-props-key
    volumes:
      - nifi-database:/opt/nifi/nifi-current/database_repository
      - nifi-flowfile:/opt/nifi/nifi-current/flowfile_repository
      - nifi-content:/opt/nifi/nifi-current/content_repository
      - nifi-provenance:/opt/nifi/nifi-current/provenance_repository
      - nifi-conf:/opt/nifi/nifi-current/conf
      - nifi-state:/opt/nifi/nifi-current/state
      - nifi-logs:/opt/nifi/nifi-current/logs
      - ./drivers:/opt/nifi/nifi-current/drivers
    networks:
      - nifi-network
    restart: unless-stopped

volumes:
  nifi-database:
  nifi-flowfile:
  nifi-content:
  nifi-provenance:
  nifi-conf:
  nifi-state:
  nifi-logs:

networks:
  nifi-network:
    driver: bridge

실행 명령

# Docker Compose 실행
docker-compose up -d

# 로그 확인
docker-compose logs -f nifi

# 중지
docker-compose stop

# 완전 제거
docker-compose down

# 볼륨까지 제거
docker-compose down -v

🛠️ 유용한 Docker 명령어

컨테이너 관리

  • 컨테이너 상태 확인: docker ps -a
  • 컨테이너 로그 확인: docker logs -f nifi
  • 컨테이너 접속: docker exec -it nifi /bin/bash
  • 컨테이너 재시작: docker restart nifi
  • 컨테이너 중지: docker stop nifi
  • 컨테이너 제거: docker rm nifi

볼륨 관리

  • 볼륨 목록 확인: docker volume ls
  • 볼륨 상세 정보: docker volume inspect nifi-database
  • 미사용 볼륨 제거: docker volume prune

네트워크 관리

  • 네트워크 목록: docker network ls
  • 네트워크 상세 정보: docker network inspect nifi-network

🔍 문제 해결

문제 원인 해결방법
NiFi UI에 접속되지 않음 초기화 중이거나 포트 충돌 1. docker logs nifi로 로그 확인
2. 포트 8443이 사용 중인지 확인
3. 1-2분 대기 후 재시도
4. HTTPS 접속시 인증서 경고는 정상 (자체 서명 인증서)
메모리 부족 오류 Docker 메모리 제한 Docker Desktop 설정에서 메모리를 4GB 이상으로 증가
또는 JVM 힙 크기 조정: -e NIFI_JVM_HEAP_INIT=512m
권한 거부 오류 볼륨 마운트 권한 문제 호스트 디렉토리 권한 변경:
chmod -R 777 ~/nifi
클러스터 노드 연결 실패 네트워크 설정 문제 1. ZooKeeper 상태 확인
2. 방화벽 규칙 확인
3. Docker 네트워크 재생성

💡 실습 예제

예제 1: CSV 파일 처리 플로우

# 샘플 데이터 디렉토리 생성
mkdir -p ~/nifi-data/input
mkdir -p ~/nifi-data/output

# 샘플 CSV 파일 생성
echo "name,age,city
John,30,New York
Jane,25,Los Angeles
Bob,35,Chicago" > ~/nifi-data/input/sample.csv

# 볼륨 마운트와 함께 NiFi 실행
docker run --name nifi-lab \
  -p 8443:8443 \
  -e SINGLE_USER_CREDENTIALS_USERNAME=admin \
  -e SINGLE_USER_CREDENTIALS_PASSWORD=adminPassword123! \
  -v ~/nifi-data:/opt/nifi-data \
  -d \
  apache/nifi:latest
실습 순서:
  1. GetFile 프로세서로 /opt/nifi-data/input 디렉토리 모니터링
  2. ConvertRecord 프로세서로 CSV를 JSON으로 변환
  3. PutFile 프로세서로 /opt/nifi-data/output에 저장

📊 모니터링 및 관리

리소스 모니터링

# 실시간 리소스 사용량 모니터링
docker stats nifi

# 상세 정보 확인
docker inspect nifi

# 디스크 사용량 확인
docker system df

# 컨테이너 프로세스 확인
docker top nifi

백업 및 복원

# 플로우 백업 (flow.xml.gz)
docker cp nifi:/opt/nifi/nifi-current/conf/flow.xml.gz ./backup-flow.xml.gz

# 플로우 복원
docker cp ./backup-flow.xml.gz nifi:/opt/nifi/nifi-current/conf/flow.xml.gz
docker restart nifi

# 전체 볼륨 백업
docker run --rm -v nifi-conf:/source -v $(pwd):/backup alpine \
  tar czf /backup/nifi-conf-backup.tar.gz -C /source .
