🚀 Apache NiFi 개요

📌 Apache NiFi란?

Apache NiFi는 시스템 간 데이터 플로우를 자동화하고 관리하기 위한 강력하고 확장 가능한 데이터 통합 플랫폼입니다. 웹 기반 UI를 통해 데이터 플로우를 시각적으로 설계, 제어, 모니터링할 수 있으며, 실시간으로 데이터를 라우팅, 변환, 시스템 중재 로직을 처리할 수 있습니다.

🎯 핵심 개념

📥 Data Ingestion → 🔄 Processing → 🔀 Routing → 💾 Data Delivery

NiFi는 데이터 수집부터 전달까지 전체 데이터 플로우를 관리합니다

✨ 주요 특징

🎨

웹 기반 UI

드래그 앤 드롭 방식의 직관적인 플로우 디자인이 가능하며, 실시간으로 데이터 플로우를 수정할 수 있습니다.

📊

데이터 프로비넌스

데이터의 전체 생명주기를 추적하고, 각 데이터가 어디서 왔고 어떻게 변환되었는지 상세히 기록합니다.

🔧

확장 가능한 아키텍처

300개 이상의 내장 프로세서를 제공하며, 커스텀 프로세서 개발을 통해 기능을 확장할 수 있습니다.

🔒

보안 기능

SSL/TLS, SSH를 통한 암호화 통신과 세밀한 접근 제어를 통해 엔터프라이즈급 보안을 제공합니다.

⚡

높은 처리량

백프레셔, 우선순위 큐잉을 통해 대용량 데이터를 효율적으로 처리할 수 있습니다.

🔄

동적 구성

시스템 중단 없이 플로우를 수정하고, 실시간으로 변경사항을 적용할 수 있습니다.

💡 사용 사례

  • IoT 데이터 수집: 수많은 IoT 디바이스로부터 실시간 데이터를 수집하고 처리
  • 로그 집계: 다양한 시스템의 로그를 중앙화하여 수집 및 분석
  • 데이터 마이그레이션: 레거시 시스템에서 최신 시스템으로 데이터 이전
  • 실시간 ETL: 데이터 추출, 변환, 적재 프로세스 자동화
  • 데이터 레이크 구축: 다양한 소스의 데이터를 데이터 레이크로 통합
  • 마이크로서비스 통합: 마이크로서비스 간 데이터 교환 및 통합

🚀 시작하기

빠른 시작 가이드

NiFi를 시작하는 가장 간단한 방법:

# NiFi 다운로드
wget https://archive.apache.org/dist/nifi/1.23.2/nifi-1.23.2-bin.tar.gz

# 압축 해제
tar -xvf nifi-1.23.2-bin.tar.gz

# NiFi 시작
./nifi-1.23.2/bin/nifi.sh start

# 웹 UI 접속
# https://localhost:8443/nifi

📚 핵심 용어 정리

📄 FlowFile

NiFi에서 처리되는 데이터의 기본 단위. 콘텐츠와 속성(메타데이터)로 구성

⚙️ Processor

데이터를 입력받아 처리하고 출력하는 NiFi의 핵심 구성 요소

🔗 Connection

프로세서 간 데이터를 전달하는 큐. 백프레셔와 우선순위 설정 가능

📦 Process Group

여러 프로세서와 연결을 그룹화하여 관리하는 논리적 컨테이너

🎛️ Controller Service

여러 프로세서가 공유하는 설정이나 리소스를 관리하는 서비스

📋 Template

재사용 가능한 플로우 구성을 저장한 파일
